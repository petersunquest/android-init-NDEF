## 根因与修复

BaseScan 编译出的 bytecode 与链上不一致（err_code_2）：
- **链上/本地 artifact**：第一个 selector `0a106e9c`
- **exportMinimalStandardJson 导出的 JSON**：BaseScan 编译后第一个 selector 为 `0d61b519`，bytecode 不同

**解决**：必须使用 **build-info 导出的 Standard JSON**（与 Hardhat 编译输入完全一致），而非 exportMinimalStandardJson。

1. 打开 https://basescan.org/address/0xc12fBEA081aD0B8143747Fd2935CE6b61734eB41#code
2. 点击 "Verify and Publish"
3. 选择 **"Solidity (Standard-Json-Input)"**
4. Compiler Version: **v0.8.33+commit.64118f21**（必须精确）
5. 上传 `deployments/base-GovernanceModule-standard-input-FULL.json`（优先，与 Hardhat 完全一致）
6. **Contract Name**：手动输入（不要从下拉选错）：
   ```
   project/src/BeamioUserCard/GovernanceModule.sol:BeamioUserCardGovernanceModuleV1
   ```
7. Constructor Arguments: **留空**
8. **关键**：若页面有 "Optimization" / "Runs" 输入框，必须设为 **Optimization: Yes, Runs: 0**（与 JSON 内 settings.optimizer.runs 一致）。若 BaseScan 从 JSON 读取则无需改。
